# RIKA Lens Calculator — Agent Guide

## Что это

Одностраничный калькулятор подбора объектива для тепловизионных прицелов RIKA NV.
Вычисляет ошибку округления px/mrad для пары сенсор + дисплей при разных фокусных расстояниях.

## Архитектура

Весь UI — один файл `src/App.tsx`. Нет роутинга, нет стора, нет отдельных компонентов.
Стили — inline. Единственный CSS-файл `src/global.css` содержит reset, тултипы и анимации.

## Правила работы с кодом

### Версионирование
- Версия отображается в заголовке: `{t("title")} <span>v3.X</span>`
- **При каждом коммите** версия инкрементируется (patch для мелких правок, minor для новых фич)
- Обновлять и в `App.tsx` (отображение), и в `README.md` (changelog)

### Тултипы таблицы
- Используется `data-tip` атрибут + CSS `::after` (определено в `global.css`)
- НЕ использовать нативный `title` — он медленный и незаметный
- Тексты тултипов должны быть **подробными** — объяснять физический смысл колонки, а не просто дублировать название

### Инпуты числовые (Nm)
- Валидация (clamp min/max) на `onBlur`, НЕ на `onChange`
- Компонент `Nm` хранит draft-строку в локальном state, коммитит при blur/Enter
- Это нужно чтобы пользователь мог свободно стирать и вводить новое значение

### Приоритет осей (SortMode)
- Три режима: `"both"` (max H,V), `"vPriority"` (сортировка V→H), `"vOnly"` (только V)
- `getScore()` вычисляет итоговую ошибку в зависимости от режима
- `sortRows()` сортирует результаты — для `vPriority` двойная сортировка V→H
- Блок `Explain` адаптирует формулу и пример под текущий режим
- В режиме `vOnly` колонки H приглушены (`C.hint`)
- Текущий режим отображается текстовой меткой рядом с заголовком таблицы результатов

### Jackpot-индикатор (ideal)
- `isIdeal` = обе оси ошибка < 0.01% (фактический ноль)
- Ideal-строки: пульсирующая звёздочка ✦ (анимация `jackpot-pulse` в `global.css`), бейдж `IDEAL` рядом с `#N`, фон `#00ff8812`
- Первая колонка: для ideal — ✦, для остальных — цветной кружок 8×8 (зелёный < 1%, жёлтый < 5%, красный > 5%)
- Ideal появляется только при совпадении аспектов сенсора и дисплея (например, 640×480 + 1024×768)

### i18n
- Три языка: `ru`, `en`, `zh` — все строки в объекте `T` в начале файла
- Переключение через `LangSw`, сохранение в `localStorage`
- При добавлении новых строк — добавлять во все три языка

### Пресеты
- `DETECTOR_PRESETS` — сенсоры: 256×192, 384×288, 640×480, 640×512, 1024×768, 1280×1024
- `DISPLAY_PRESETS` — дисплеи: 640×480, 1024×768, 1280×1024, 1920×1080, 2560×2560
- `PITCH_OPTIONS` — пиксельный шаг: 12, 15, 17, 25 µm
- Новые пресеты добавлять в конец массива, не менять индексы существующих (ломает `useState` дефолты)
- Дефолтный индекс `dI` должен соответствовать нужному сенсору после добавления

## Что НЕ делать

- Не выносить компоненты в отдельные файлы
- Не добавлять CSS-фреймворки
- Не менять `base` в `vite.config.ts` (привязан к имени репо)
- Не обрезать тексты тултипов — они должны быть подробными
- Не заменять `data-tip` на `title`
