# RIKA Lens Calculator — Agent Guide

## Что это

Одностраничный калькулятор подбора объектива для тепловизионных прицелов RIKA NV.
Вычисляет ошибку округления px/mrad для пары сенсор + дисплей при разных фокусных расстояниях.

## Архитектура

Весь UI — один файл `src/App.tsx`. Нет роутинга, нет стора, нет отдельных компонентов.
Стили — inline. Единственный CSS-файл `src/global.css` содержит reset, тултипы и анимации.

## Правила работы с кодом

### Версионирование
- Версия отображается в заголовке: `{t("title")} <span>v3.X</span>`
- **При каждом коммите** версия инкрементируется (patch для мелких правок, minor для новых фич)
- Обновлять и в `App.tsx` (отображение), и в `README.md` (changelog)

### Тултипы таблицы
- Заголовки колонок используют нативный `title` атрибут (через TH компонент)
- `tip()` — алиас для `t()`, используется для тултипов по ключам `tipPixelSize`, `tipPosError`, `tipModeBoth` и др.
- Тексты тултипов должны быть **подробными** — объяснять физический смысл колонки, а не просто дублировать название

### Раскрывающиеся строки (expanded)
- `expanded` state хранит `number | null` (F value раскрытой строки)
- Клик по строке таблицы → toggle expand, НЕ добавление в сравнение
- Раскрытая строка: `<tr><td colSpan={9}>` с двумя таблицами дистанций + кнопка сравнения
- Кнопка «Добавить в сравнение» с `e.stopPropagation()` чтобы не сворачивать строку
- Индикатор ▸/▾ в первой ячейке, borderLeft green для раскрытой строки

### Инпуты числовые (Nm)
- Валидация (clamp min/max) на `onBlur`, НЕ на `onChange`
- Компонент `Nm` хранит draft-строку в локальном state, коммитит при blur/Enter
- Это нужно чтобы пользователь мог свободно стирать и вводить новое значение

### Приоритет осей (SortMode)
- Три режима: `"both"` (max H,V), `"vPriority"` (сортировка V→H), `"vOnly"` (только V)
- `getScore()` вычисляет итоговую ошибку в зависимости от режима
- `sortRows()` сортирует результаты — для `vPriority` двойная сортировка V→H
- Блок `Explain` адаптирует формулу и пример под текущий режим
- В режиме `vOnly` колонки H приглушены (`C.hint`)
- Текущий режим отображается текстовой меткой рядом с заголовком таблицы результатов

### Jackpot-индикатор (ideal)
- `isIdeal` = обе оси ошибка < 0.01% (фактический ноль)
- Ideal-строки: пульсирующая звёздочка ✦ (анимация `jackpot-pulse` в `global.css`), бейдж `IDEAL` рядом с `#N`, фон `#00ff8812`
- Первая колонка: ✦ только для ideal-строк, пустая для остальных
- Ideal появляется только при совпадении аспектов сенсора и дисплея (например, 640×480 + 1024×768)

### URL sharing
- Single: `tab=single&det=&disp=&pitch=&from=&to=&mode=&lang=` в `window.location.hash`
- Lineup: `tab=lineup&from=&to=&mode=&agg=&thr=&lang=&c1=detI,pitchI,dispI&...`
- При загрузке: парсится hash → используется как начальные значения useState
- При изменении параметров: hash обновляется через `useEffect`
- `parseHash()` и `_hp` определены на уровне модуля — вызываются один раз при загрузке
- Кнопка «Скопировать ссылку» рядом с переключателем языка, обратная связь «✓ Скопировано» на 2 сек

### Вкладки (Tab)
- Два режима: `"single"` (один прибор) и `"lineup"` (продуктовая линейка)
- Состояние в `tab` state, сохраняется в URL hash (`tab=single` / `tab=lineup`)
- Вкладки — кнопки в стиле SortMode (minimal, активная с highlight + border)
- Single: вся оригинальная функциональность без изменений
- Lineup: подбор объективов для нескольких конфигураций приборов

### Продуктовая линейка (Lineup)
- `lCfg: LConfig[]` — до 6 конфигураций (мин. 2), каждая: name, detI, pitchI, dispI
- `lFF, lFT` — диапазон фокусных (общий для всех конфигураций)
- `agg: AggMode` — режим агрегации: `"max"` / `"avg"` / `"coverage"`
- `thr` — порог для режима coverage (0.1–10%)
- `lExp` — раскрытая строка lineup-таблицы (F value)
- Вычисления: `lResults` → `lSorted` → `lTop5` (аналогично single)
- Таблица: колонки `#, F, Config1%, Config2%, ..., Aggregate, Coverage`
- Раскрытая строка: для каждой конфигурации — px/mrad, ошибки, таблицы дистанций
- URL hash для lineup: `tab=lineup&from=&to=&mode=&agg=&thr=&lang=&c1=detI,pitchI,dispI&c2=...`
- Цвета конфигураций: `CMP_COLORS[ci]`

### Сравнение конфигураций (Compare)
- Добавление в сравнение — через кнопку в раскрытой строке (макс. 9)
- Сравнённые строки получают `borderLeft: 3px solid` с цветами `CMP_COLORS`
- Панель сравнения — между таблицей и блоком «Описание колонок»
- Каждая карточка содержит: px/mrad, ошибки, итог + таблицу размера пикселя по дистанциям (100–1000 м)
- `DISTANCES = [100, 200, 300, 500, 700, 1000]`
- Формула: `mm100 * D / 100` — линейный размер 1 пикселя на дистанции D

### i18n
- Три языка: `ru`, `en`, `zh` — все строки в объекте `T` в начале файла
- Переключение через `LangSw`, сохранение в `localStorage`
- При добавлении новых строк — добавлять во все три языка

### Пресеты
- `DETECTOR_PRESETS` — сенсоры: 256×192, 384×288, 640×480, 640×512, 1024×768, 1280×1024
- `DISPLAY_PRESETS` — дисплеи: 640×480, 1024×768, 1280×1024, 1920×1080, 2560×2560
- `PITCH_OPTIONS` — пиксельный шаг: 12, 15, 17, 25 µm
- Новые пресеты добавлять в конец массива, не менять индексы существующих (ломает `useState` дефолты)
- Дефолтный индекс `dI` должен соответствовать нужному сенсору после добавления
